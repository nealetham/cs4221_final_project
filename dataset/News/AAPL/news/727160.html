Five Languages That Won’t Ever Die

Immortal is a word I’d use to describe them

Kell

·

When it comes to programming, software development/engineering, and all of the enclosed fields, you see that, in the real world, there are really only a handful of languages that get used in day to day operations, and it is extremely hard for any one language to break that barrier to entry.

There are some very good reasons as to why the popularity of programming languages doesn’t shift all that often, and the biggest one, in my eyes, is backwards compatibility with existing code bases. The biggest problem with introducing a new language into the ecosystem is that developers have a choice to make: find a way to make it work with what they have, or start over fresh…and that’s not a fun choice to make.

Also, please note that my opinions are going to be sprinkled into this story about as much as possible because it’s just not fun writing a story like this one without throwing some anecdotal and potentially arbitrary comments into the mix.

With all that in mind, let’s look at five programming languages that just won’t die. Ever. Like, ever ever.

C / C++

Honestly, I like these two languages. They’re old, but still viable for use in more ways than one. C++ is kept updated by the ISO committee dedicated to it, and it just feels modern, although, that can change the second you get a look at an older code base or one that doesn’t make use of the newer standards.

They are the undisputed kings of the world. The need for backwards compatibility has these big guys sitting at the top of a very, very steep pyramid. C and C++ have been around for nearly 40 (50 in C’s case) years, and they show little to no signs of slowing down any time soon.

The reason they’re so popular, as said above, is mainly a need for backwards compatibility with legacy code bases. And the reason that I put these two together is because they’re so closely interoperable, and that’s mostly because C++ was designed with that in mind. Backwards compatibility is king.

Python

I genuinely despise this language, mostly for its extremely noisy and, honestly, dirty syntax, but I have to give credit where credit is due, because Python, over the last thirty years, has become a juggernaut of genuinely, at least in my opinion, awesome proportions.

Like C and C++, though, Python remains popular because of the need for backwards compatibility, but Python also has simplicity and ease of use on its side, which the aforementioned two do not.

Python’s low barrier to entry makes it the perfect choice for the beginner and often makes it fun to use. This is probably the biggest reason it won’t die, in my opinion: its ease of use.

Java

I don’t despise this language, but I definitely don’t like it. When I look at Java code, I often cringe as it is just a chore to read without good documentation paired with it, and that’s rarer than I’d like it to be, honestly.

Despite my reservations about it, however, Java maintains a consistent top-five position on the TIOBE index, which means that a lot of people must like it for one reason or another. I just don’t understand how anyone can like this language for just about anything when there are newer alternatives that accomplish Java’s goals in a much nicer looking fashion.

Personally, I find that Java is often bloated and a chore to read, as I said above. It’s just hard for me to enjoy reading or writing it.

Swift

I really didn’t want to put this guy on here, and I’m betting at least some of you are confused as to why I did, so let me explain myself.

Swift was introduced by Apple, Inc. with the sole purpose of replacing Objective-C, which I had considered putting on this list and ended up putting it below (in the “honorable mentions” category). Swift, having been created and backed by Apple, isn’t going to die — at least not anytime soon — because of that affiliation.

It’s also a main competitor for spaces occupied by the likes of C, C++, and Objective-C, and even Rust, depending on who you talk to (I’m biased, sorry). This is because Swift is designed to work, like the previously mentioned names, at a lower level than languages like Java or C#, and this means that it’s bound to have some sort of a following as it’s useful in more than just front-end application development.

Honorable Mentions

Here are some languages that I think have potential, but aren’t necessarily “immortal”.

Rust

I had to. I mean, come on. Would this be my list without Rust being on it? Absolutely not! I’m quite the fan of Rust and I’ll pledge allegiance to it until the day I kick the bucket.

The only reason that Rust isn’t placed on the main portion of this list is because it’s relatively young and doesn’t have the backing of someone like Apple, although Microsoft has recently put forth some apparent endorsement in the form of an official WinRT language projection. So, there’s that.

C#

I would put this higher, and, honestly, I might still, but I just don’t see as much widespread interest in C# the way that I do for something like Java. It just isn’t there, at least in my sphere.

Objective-C

The predecessor to Swift. This language probably won’t be dying anytime soon, given the need for backwards compatibility, as mentioned in the paragraphs underneath C and C++, but I don’t see it living for too much longer, given the fact that it’s more or less been forcibly deprecated by Apple in an effort to modernize.

PHP

PHP Hypertext Processor has remained a constant in the top ten on the TIOBE index for the last twenty years. It’s continued use in web development makes it a titan of this industry.

Summary

So, obviously, I’m not all-knowing and I can’t predict what languages are truly immortal, but this was my “best” shot while also trying to be half-way entertaining to any reader.

I appreciate your interest in my story, and I thank you for reading this far! Have a wonderful day! :)
